# Contributing to Smart Loan Recovery System

Thanks for your interest in contributing! Below are guidelines to make contributions smooth.

## How to contribute
1. Fork the repository.
2. Create a branch: git checkout -b feature/your-feature-name.
3. Make your changes and add tests where appropriate.
4. Open a pull request describing your changes.

## Coding style
- Python code should follow PEP8.
- Use docstrings for all functions and classes.
- Write unit tests for new features.

## Testing
- Tests should be placed in tests/.
- Run tests with pytest.

## Reporting issues
- Open issues with a clear title and steps to reproduce.
- Label issues (bug, enhancement, question).
